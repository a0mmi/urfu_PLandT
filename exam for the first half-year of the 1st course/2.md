## Лекция 2. Типы данных и приведение типов в C++

**Базовые типы данных:** В C++ есть несколько категорий типов.  **Целочисленные** : `char`, `short`, `int`, `long`, `long long` (со знаком и без знака). Размеры зависят от платформы (минимально `char` 1 байт, `short` ≥2, `int` ≥2, `long` ≥4, `long long` ≥8 байт). Примеры диапазонов: `int` обычно –2³¹…2³¹–1 (для 32-бит). `char` – 1 байт, может быть либо знаковый, либо беззнаковый (этим и объясняется зависимость диапазона от реализации). Есть также спецтипы: `bool` (1 байт, значение `true`/`false`), указатель `void*`, которые не арифметические.

**Символьные типы:** `char` хранит один символ (обычно ASCII/UTF-8); также есть `wchar_t`, `char16_t`, `char32_t` для широких символов (широкие строки). `char` в памяти – однобайтовый, то есть нельзя хранить символы Unicode >255 напрямую.

**Вещественные типы:** `float` (обычно 32 бита), `double` (64 бита) и `long double` (128 бит или 80-битный в зависимости от системы) хранят числа с плавающей точкой. Пример: на большинстве систем `float` ~1.5·10^−45…3.4·10^38, `double` ~5.0·10^−324…1.7·10^308. Вещественные литералы по умолчанию имеют тип `double`; суффиксы `f` или `l` задают `float` или `long double` (например, `3.14f` – float).

**Другие типы:** `enum` – перечислимый тип, хранит целочисленное значение (номер константы). `struct`, `class` и `union` – пользовательские типы. Массивы и указатели (в том числе `std::string` и другие объекты) технически являются составными.

C++ – статически типизированный язык: тип переменной задаётся при объявлении и не меняется во время работы программы. Например:

```C++
int a = 5;       // a – int
double x = 2.0;  // x – double
```

**Неявное приведение (conversion):** При смешанных операциях C++ автоматически преобразует типы по ряду правил. К примеру, если складываем `int + double`, `int` преобразуется в `double` (результат – `double`). При присвоении значений без приведения типов компилятор либо неявно конвертирует, либо выдаёт предупреждение/ошибку. Например:

```C++
unsigned int u = -25;  // ошибка: «narrowing conversion», компилятор не допустит такой присвоение:contentReference[oaicite:16]{index=16}.
```

Правило: при присвоении константы отрицательного типа `int` беззнаковому `unsigned` происходит ошибка компиляции (узкий конверт).

Список приоритетов при приведении числовых типов при операциях: при вычислении `(long double, double, float, unsigned long long, long long, unsigned long, long, unsigned int, int)` – типы более высокого в списке расширяют более низкие. Это значит, например, что при операции `int + double` результат будет `double`, а не `int`.

**Явное приведение:** Для контроля преобразований используют операторы-конверсии: `static_cast`, `dynamic_cast`, `const_cast`, `reinterpret_cast`. Чаще всего – `static_cast` для приведения арифметических типов или наследования. Например:

```C++
double d = 7.3;
int n = static_cast<int>(d);  // n=7, дробная часть отброшена
```

Пример: при делении `unsigned sum / hours` дробная часть отбрасывалась бы, но с `static_cast` можно явно преобразовать результат к `unsigned int`. `static_cast` выполняет преобразование проверяемое во время компиляции. В старом C-синтаксисе можно было писать `(int)d`, но современный подход – `static_cast`.

**Сравнение указателей и ссылок:** Указатель (`Type*`) хранит адрес объекта, может быть `nullptr`, его можно переназначать. Ссылка (`Type&`) – это «имя» для уже существующего объекта, всегда должно ссылаться на что-то существующее, её нельзя «обнулить» или перенаправить. При передаче параметров в функцию «по ссылке» функция получает alias исходной переменной и может изменить её значение (см. Лекцию 8).

**Практическое применение:** Важно выбирать минимально достаточный тип для экономии памяти и предотвращения переполнения. Например, для флагов или маленьких чисел предпочтителен `int` или `char`, для больших счётчиков – `long long`. Тип `bool` экономит пространство, но при арифметике ведёт себя как число 0/1. Используйте `signed`/`unsigned` осознанно: беззнаковый полезен для счётчиков и битовых операций. Для безопасного кодирования приводите типы явно там, где есть риск потери данных или предупреждений – это делает намерение очевидным для компилятора.
