## Лекция 2. Типы данных и приведение типов в C++

**Базовые типы данных:** В C++ есть несколько категорий типов.  **Целочисленные** : `char`, `short`, `int`, `long`, `long long` (со знаком и без знака). Размеры зависят от платформы (минимально `char` 1 байт, `short` ≥2, `int` ≥2, `long` ≥4, `long long` ≥8 байт). Примеры диапазонов: `int` обычно –2³¹…2³¹–1 (для 32-бит). `char` – 1 байт, может быть либо знаковый, либо беззнаковый (этим и объясняется зависимость диапазона от реализации). Есть также спецтипы: `bool` (1 байт, значение `true`/`false`), указатель `void*`, которые не арифметические.

**Символьные типы:** `char` хранит один символ (обычно ASCII/UTF-8); также есть `wchar_t`, `char16_t`, `char32_t` для широких символов (широкие строки). `char` в памяти – однобайтовый, то есть нельзя хранить символы Unicode >255 напрямую.

**Вещественные типы:** `float` (обычно 32 бита), `double` (64 бита) и `long double` (128 бит или 80-битный в зависимости от системы) хранят числа с плавающей точкой. Пример: на большинстве систем `float` ~1.5·10^−45…3.4·10^38, `double` ~5.0·10^−324…1.7·10^308. Вещественные литералы по умолчанию имеют тип `double`; суффиксы `f` или `l` задают `float` или `long double` (например, `3.14f` – float).

**Другие типы:** `enum` – перечислимый тип, хранит целочисленное значение (номер константы). `struct`, `class` и `union` – пользовательские типы. Массивы и указатели (в том числе `std::string` и другие объекты) технически являются составными.

**Диапозоны:**
| Тип данных              | Диапазон значений                      | Размер (байт) |
|-------------------------|----------------------------------------|---------------|
| `char` (signed)         | −128 … 127                             | 1             |
| `unsigned char`         | 0 … 255                                | 1             |
| `short int`             | −32 768 … 32 767                       | 2             |
| `unsigned short int`    | 0 … 65 535                             | 2             |
| `int`                   | −32 768 … 32 767                       | 2             |
| `unsigned int`          | 0 … 65 535                             | 2             |
| `long int`              | −2 147 483 648 … 2 147 483 647         | 4             |
| `unsigned long int`     | 0 … 4 294 967 295                      | 4             |
| `float`                 | ≈ −3.4×10³⁸ … 3.4×10³⁸                 | 4             |
| `double`                | ≈ −1.7×10³⁰⁸ … 1.7×10³⁰⁸               | 8             |
| `long double`           | ≈ −1.7×10⁴⁹³² … 1.7×10⁴⁹³²             | 10            |


C++ – статически типизированный язык: тип переменной задаётся при объявлении и не меняется во время работы программы. Например:

```C++
int a = 5;       // a – int
double x = 2.0;  // x – double
```

## Сравнение (целые типы)
- При сравнении типов одинакового знака и размера — всё просто: сравниваем значения.
- При сравнении **signed** и **unsigned** действует правило: либо signed приводится к unsigned (если ранк unsigned ≥ ранк signed), что может сделать отрицательное значение большим положительным в представлении unsigned. Это частый источник ошибок.
- Если размеры различаются, происходит приведение к типу с большим рангом (шире) согласно правилам языка.
- Рекомендации:
  - Избегать смешивания signed/unsigned в сравнении.
  - При необходимости — явно приводить тип (`static_cast`) к ожидаемому типу перед сравнением.
  - Для счётчиков и индексов обычно используют `std::size_t` или `unsigned` (но при сравнении с signed будьте аккуратны).

Пример «подводного камня»:
```c++
int a = -1;
unsigned int b = 1u;
if (a < b) { /* условие ложно, потому что a приведён к unsigned */ }
```

## Сравнение вещественных типов
- Прямая проверка на равенство (`==`) ненадёжна из‑за ошибок округления при представлении чисел с плавающей точкой.
- Использовать сравнение с эпсилон (абсолютная или относительная погрешность):
```c++
bool almost_equal_abs(double a, double b, double eps = 1e-9) {
    return std::fabs(a - b) <= eps;
}

bool almost_equal_rel(double a, double b, double rel_eps = 1e-9) {
    return std::fabs(a - b) <= rel_eps * std::max(std::fabs(a), std::fabs(b));
}
```
- Обрабатывать `NaN` и `inf` отдельно: `NaN != NaN`. Для проверки используйте `std::isnan`, `std::isinf`.
- При сортировке или сравнениях больших массивов вещественных значений учитывайте накопление ошибок и численные нестабильности алгоритмов.

## Присваивание
- При `T x = e;` выражение `e` неявно преобразуется к типу `T` (если возможно).
- Списковая инициализация `T x{e};` чаще запрещает narrowing-конверсии (что безопаснее).
- Примеры опасных присвоений:
```c++
unsigned u = -5;      // результат — большое значение unsigned (по модулю 2^N)
int8_t small = 300;   // narrowing — потеря данных (значение не помещается)
double d = 1e100;     // при присвоении в float возможна потеря точности/переполнение
```
- Рекомендация: использовать `{}` при инициализации констант и там, где требуется запрет narrowing; в остальных местах явно приводить тип, если есть сомнения.

**Неявное приведение (conversion):** При смешанных операциях C++ автоматически преобразует типы по ряду правил. К примеру, если складываем `int + double`, `int` преобразуется в `double` (результат – `double`). При присвоении значений без приведения типов компилятор либо неявно конвертирует, либо выдаёт предупреждение/ошибку. Например:

```C++
unsigned int u = -25;  // ошибка: «narrowing conversion», компилятор не допустит такой присвоение:contentReference[oaicite:16]{index=16}.
```

Правило: при присвоении константы отрицательного типа `int` беззнаковому `unsigned` происходит ошибка компиляции (узкий конверт).

Список приоритетов при приведении числовых типов при операциях: при вычислении `(long double, double, float, unsigned long long, long long, unsigned long, long, unsigned int, int)` – типы более высокого в списке расширяют более низкие. Это значит, например, что при операции `int + double` результат будет `double`, а не `int`.

**Явное приведение:** Для контроля преобразований используют операторы-конверсии: `static_cast`, `dynamic_cast`, `const_cast`, `reinterpret_cast`. Чаще всего – `static_cast` для приведения арифметических типов или наследования. Например:

```C++
double d = 7.3;
int n = static_cast<int>(d);  // n=7, дробная часть отброшена
int a = (int)d // Тоже явное
```

Пример: при делении `unsigned sum / hours` дробная часть отбрасывалась бы, но с `static_cast` можно явно преобразовать результат к `unsigned int`. `static_cast` выполняет преобразование проверяемое во время компиляции. В старом C-синтаксисе можно было писать `(int)d`, но современный подход – `static_cast`.

## Переполнение типов
- **Signed integer overflow** (переполнение целых со знаком) — *undefined behavior* (UB). Нельзя полагаться на поведение в таком случае.
- **Unsigned integer overflow** — определено как арифметика по модулю 2^N (wrap-around).
- **Floating point overflow** — даёт `+inf`/`-inf` и флаги IEEE 754; underflow может привести к денормализованным числам или нулю.
- **Защита:** проверяйте диапазоны заранее, используйте более широкий тип (`long long`, `long double`), или библиотечные/компиляторные средства для детекции переполнений (санитайзеры, проверки вручную).

Пример проверки до сложения (для положительных чисел):
```c++
bool will_add_overflow(unsigned int a, unsigned int b) {
    return a > std::numeric_limits<unsigned int>::max() - b;
}
```

## Расширение и сужение диапазонов (widening / narrowing)
- **Расширение (widening):** преобразование от узкого к широкому типу (например, `int` → `long long`) — обычно безопасно (информация не теряется при сохранении знака).
- **Сужение (narrowing):** преобразование от широкого к узкому типу (например, `long long` → `int`) — потенциально опасно, может обрезать старшие биты и изменить знак.
- **Списковая инициализация** (`T x{v};`) в C++ предотвращает некоторые виды narrowing на этапе компиляции (если значение не помещается, компилятор выдаст ошибку).
- **Рекомендации:** при преобразованиях, где есть риск потери данных, явно используйте `static_cast` и добавляйте проверки диапазона до приведения.
