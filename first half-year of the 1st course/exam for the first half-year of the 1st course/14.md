# 14. Вычислительная сложность алгоритмов. О-нотация. Основные показатели сложности (линейная, константная, квадратичная и т.д.) и их характеристики. Примеры алгоритмов с заданной сложностью. Понятие лучшего, худшего, среднего случая.

## Введение
Вычислительная (временная) сложность алгоритма — это оценка того, как меняется время выполнения алгоритма в зависимости от размера входных данных `n`. Цель — описать поведение алгоритма при больших `n`, игнорируя детали реализации, аппаратные отличия и константы.
Помимо временной сложности, существует **пространственная сложность** — оценка используемой дополнительной памяти.

## Блоки: О-нотация и другие нотации
### Основные обозначения
-**Big-O (O)** — верхняя граница (asymptotic upper bound). Говорим `T(n) = O(f(n))`, если для достаточно больших `n` время `T(n)` не больше константы умноженной на `f(n)`.
-**Big-Theta (Θ)** — точная асимптотическая граница: `T(n) = Θ(f(n))`, если `T(n)` одновременно `O(f(n))` и `Ω(f(n))`.
-**Big-Omega (Ω)** — нижняя граница (asymptotic lower bound): `T(n) = Ω(f(n))`.
-**little-o (o)** и **little-omega (ω)** — строгие варианты: `T(n) = o(f(n))` означает `T(n)/f(n) -> 0` при `n->∞`.

### Формальное определение Big-O

Функция `T(n)` принадлежит `O(f(n))`, если существуют `C > 0` и `n0` такие, что для всех `n >= n0` выполняется:

```
T(n) <= C * f(n)

```

## Основные классы сложности и их свойства

Ниже — стандартная таблица с порядками роста (от быстрого к медленному в смысле роста ресурса):

| Нотация | Описание | Пример роста при увеличении n |
|--------|---------|------------------------------|
| `O(1)`  | Константная | не меняется |
| `O(log n)` | Логарифмическая | растёт очень медленно |
| `O(n)`  | Линейная | пропорционально n |
| `O(n log n)` | Лин.-лог. | типично для эффективных сортировок |
| `O(n^2)` | Квадратичная | быстро растёт (матрицы, простая сортировка) |
| `O(n^3)` | Кубическая | более ресурсоёмкая |
| `O(2^n)` | Экспоненциальная | растёт очень быстро |
| `O(n!)` | Факториальная | практически не применима для больших n |

## Характеристики классов (коротко)

-**O(1)** — время не зависит от `n`. Пример: доступ к элементу массива по индексу.

-**O(log n)** — деление задачи на части (бинарный поиск), двоичные деревья.

-**O(n)** — один простой проход по данным: суммирование массива.

-**O(n log n)** — оптимальные сравнивающие сортировки (quicksort—в среднем, mergesort, heapsort).

-**O(n^2)** — вложенные двойные циклы: сравнение каждой пары элементов (bubble sort, selection sort).

-**O(2^n)** — перебор всех подмножеств (subset enumeration).

-**O(n!)** — полный перебор перестановок (bruteforce TSP naive).


## Примеры алгоритмов с указанной сложностью
-`O(1)`: доступ `A[i]`, присваивание `x = 5`, `push` в фиксированную структуру.
-`O(log n)`: бинарный поиск в отсортированном массиве.
-`O(n)`: линейный поиск, подсчёт суммы, однопроходный подсчёт.
-`O(n log n)`: `std::sort` (реализация intro/mergesort/quicksort), heapsort, mergesort.
-`O(n^2)`: bubble sort, insertion sort (в худшем случае), naive matrix multiplication (тройной вложенный цикл — `O(n^3)`).
-`O(n^3)`: стандартная матричная умножение `n x n`.
-`O(2^n)`: перебор всех подмножеств, naive решения для задач о рюкзаке перебором битмасок.
-`O(n!)`: перебор всех перестановок.


## Как вычислять сложность: приёмы и правила

### 1. Правило суммы
Если алгоритм выполняет две последовательные части с временем `T1(n)` и `T2(n)`, то суммарно `T(n) = T1(n) + T2(n)`. Асимптотически берём максимум: `O(max(T1,T2))`.

### 2. Правило произведения
Если части вложены (одна внутри другой), `T(n) = T1(n)*T2(n)`.

### 3. Игнорируем константы и низкоуровневые детали
`O(3n)` эквивалентно `O(n)`; `O(n + 1000)` -> `O(n)`.

### 4. Удаляем меньшие слагаемые
`O(n^2 + n)` -> `O(n^2)`.

### 5. Циклы
- Один цикл `for i in 1..n` → `O(n)`.
- Два последовательных цикла → `O(n) + O(n) = O(n)`.
- Вложенные два цикла `for i in 1..n``for j in 1..n` → `O(n^2)`.
- Вложенный `for j in i..n` → `O(n^2)` (количество итераций ~ n(n+1)/2).

### 6. Рекурсия
Анализируем рекурренту. Пример: `T(n) = 2T(n/2) + O(n)` (merge sort) → `T(n) = O(n log n)` (Master Theorem).

### 7. Master Theorem (коротко)
Для рекуррент `T(n) = a T(n/b) + f(n)`:
- Если `f(n) = O(n^{log_b a - ε})` → `T(n) = Θ(n^{log_b a})`.
- Если `f(n) = Θ(n^{log_b a} * log^k n)` → `T(n) = Θ(n^{log_b a} * log^{k+1} n)`.
- Если `f(n) = Ω(n^{log_b a + ε})` и регулярность → `T(n) = Θ(f(n))`.

Примеры:
- Merge sort: `a=2, b=2, f(n)=Θ(n)` → `n^{log_2 2} = n` → второй случай → `Θ(n log n)`.
- Binary tree traversal: `T(n) = 2T(n/2) + O(1)` → `Θ(n)`.

## Лучший, худший и средний случаи
-**Лучший случай (best-case)** — поведение на самом «легком» входе. Пример: insertion sort `O(n)` при уже отсортированном массиве.
-**Худший случай (worst-case)** — поведение на самом «плохом» входе; важен для гарантий корректности и ресурсов. Пример: quicksort `O(n^2)` в худшем случае (в случае плохого выбора опорного элемента, например уже отсортированный массив и выбор первого элемента как pivot).
-**Средний случай (average-case)** — ожидаемая сложность по распределению входов (упирается в модель случайности). Например, quicksort `O(n log n)` в среднем при случайных pivot или случайном порядке входа.
Важно: при доказательстве корректности и оценки алгоритмов часто ориентируются на худший случай для жестких гарантий, но средняя/амортизированная сложность важна для практики.


## Амортизированная сложность
Амортизированная оценка даёт среднюю стоимость операции по последовательности операций, даже если отдельная операция дорогая.
**Пример:**`vector.push_back` с удвоением размера:
- Большинство вставок — `O(1)`.
- Периодически происходит `realloc` и копирование — `O(n)` для одной операции.
- Амортизированная стоимость одной вставки — `O(1)` (потому что перераспределение происходит редко и его стоимость распределяется на множество вставок).


## Пространственная (memory) сложность
Оценивается дополнительная память, независимая от входа (не считая памяти для самих входных данных, если не требуется).
-`O(1)` — константная доп. память.
-`O(n)` — линейная доп. память.
Пример: merge sort требует `O(n)` дополнительной памяти (в простом варианте), quicksort — `O(log n)` стека рекурсии в среднем (если реализация in-place).

## Частые ошибки при оценке сложности
1.**Путаница между `O(1)` и `O(n)`**: доступ к элементу массива — `O(1)`, поиск по unsorted array — `O(n)`.
2.**Игнорирование вложенных циклов**: вложенные циклы часто дают произведение сложностей.
3.**Сравнение `int` и `unsigned`** — не относится к сложности, но частая логическая ошибка в коде.
4.**Сложность по времени ≠ сложность по памяти**.
5.**Асимптотика = поведение при больших n** — при малых n константы могут доминировать.

## Примеры анализа (пошагово)
### Пример 1: простой цикл
```pseudo
sum = 0
for i = 1 to n:
    sum += a[i]
```
Итераций `n` → `O(n)`.

### Пример 2: вложенные циклы
```pseudo
for i = 1 to n:
    for j = 1 to n:
        do O(1) work
```
Всего `n * n` операций → `O(n^2)`.

### Пример 3: треугольные вложенные циклы
```pseudo
for i = 1 to n:
    for j = i to n:
        do O(1)
```
Итераций ~ n(n+1)/2 → `O(n^2)`.

### Пример 4: логарифмический цикл
```pseudo
i = 1
while i <= n:
    i = i * 2
```
i принимает значения 1,2,4,...,2^k <= n → k = floor(log2 n) → `O(log n)`.

### Пример 5: рекурсия (фибоначчи naive)

```pseudo
fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)
```
Рекуррент: `T(n) = T(n-1) + T(n-2) + O(1)` → `T(n) = Θ(φ^n)` (экспоненциально, где φ ≈ 1.618).

### Пример 6: быстрый бинарный поиск
```pseudo
binary_search(arr, x):
    lo = 0; hi = n-1
    while lo <= hi:
        mid = (lo + hi) / 2
        if arr[mid] == x: return mid
        if arr[mid] < x: lo = mid + 1
        else hi = mid - 1
```
Кажная итерация делит интервал пополам → `O(log n)`.

## Сравнение функций роста (пример порядка)
Для больших `n`:
```
1 << log log n << log n << sqrt(n) << n << n log n << n^2 << n^3 << 2^n << n!
```
(Здесь `<<` означает «растёт гораздо медленнее, чем».)
